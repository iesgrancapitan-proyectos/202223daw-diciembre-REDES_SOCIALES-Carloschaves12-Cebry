## Anexo Referencia Sintaxis JSON

El uso de ficheros __JSON__ para la configuración de la aplicación hace que sea necesario prestarle atención en este trabajo. En este anexo se explica la estructura de los ficheros __JSON__ y cómo se deben rellenar para que la aplicación funcione correctamente.


Un fichero __JSON__ es un fichero de texto plano que contiene una estructura de datos en formato determinado. Las cadenas de texto deben ir entre comillas dobles. La estructura de datos es un objeto que contiene una serie de pares clave-valor. La clave es un texto que identifica el valor. El valor puede ser un texto, un número, un booleano, un objeto o un array. 

Los objetos vendrán definidos con la siguiente sintaxis:

```
{
    "clave_1": valor_1,
    "clave_2": valor_2,
    "clave_3": valor_3
}
```

Debe tenerse en cuenta que las claves y los valores deben ir separados por dos puntos. Los pares clave-valor deben ir separados por comas y nunca deben terminar en una coma.

Un array es una lista de valores, que pueden ser de cualquier tipo: texto, número, booleano, objeto o array.
Los arrays vendran definidos con la sintaxis:

```
[
    valor_1,
    valor_2,
    valor_3
]
```

Los valores deben ir separados por comas y nunca deben terminar en una coma.
